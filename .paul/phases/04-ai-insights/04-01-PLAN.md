<objective>

## Goal

Build a vendor-specific AI Insights Dashboard powered by Gemini 2.5 Flash that analyzes real-time channel data and delivers actionable business intelligence — moving beyond raw metrics to meaningful, contextual insights.

## Purpose

Vendors need more than dashboards of numbers. They need to understand *what* their data means: which products are underperforming, why channel A outperforms B, when to reorder, where to optimize pricing. This plan connects live Supabase data to Gemini 2.5 Flash to generate vendor-specific, context-aware insights.

## Output

- Gemini API integration module with streaming support
- Insight calculation engine that transforms raw metrics into actionable intelligence
- Custom vendor dashboard with insight cards (performance, channels, inventory, revenue, trends, recommendations)
- Checkpoint for human verification of insight quality and dashboard UX

</objective>

<acceptance_criteria>

## AC-1: Gemini API Integration
```gherkin
Given a vendor is authenticated
When the insights dashboard loads
Then Gemini 2.5 Flash API is called with current channel data (products, sales, orders)
And response streams to the UI in real-time
And errors are caught and displayed gracefully
```

## AC-2: Insight Quality — Not Raw Values
```gherkin
Given vendor has multi-channel sales data
When insights are generated
Then insights include context (e.g., "Product X is trending 15% weekly growth in Shopify" NOT just "100 units sold")
And insights reference multiple channels for comparison
And insights include recommendations (reorder, price adjust, promote)
And each insight is independently actionable
```

## AC-3: Custom Vendor Dashboard Layout
```gherkin
Given a vendor navigates to /insights
When the dashboard loads
Then displays 4-5 insight cards with clear hierarchy
And cards show metrics + supporting context + action items
And insight cards update when channel data changes
And dashboard is responsive (mobile, tablet, desktop)
```

## AC-4: Meaningful Insight Types
Dashboard must surface insights across these dimensions (at least 3 of 5):
- **Performance:** Top products, best-selling channels, growth rates
- **Channel Comparison:** Which channel sells more, margin differences, velocity
- **Inventory Health:** Stock levels, stockout risk, recommended reorder quantities
- **Revenue:** Total by channel, top-selling products, margin analysis
- **Trends & Forecasts:** Seasonal patterns, demand shifts, AI recommendations

</acceptance_criteria>

  <files>src/lib/insight-engine.ts, src/lib/queries.ts (extend), src/types/insights.ts</files>
  <action>
    Create insight engine that:
    - Fetches current vendor's channel data (products, sales, orders, settlements from Supabase)
    - Calculates base metrics: total revenue, top 5 products, channel comparison, growth rates, inventory levels
    - Groups by insight type: performance, inventory, revenue, trends
    - Formats each group as a structured prompt for Gemini: "Here's vendor X's channel data. What are the top 3 performance insights?"
    - Calls streamInsight() from gemini-client for each insight type
    - Returns array of InsightCard objects: { title, context, metric, recommendation, insightType, timestamp }
    
    Example output:
    {
      title: "Shopify Leading Channel",
      context: "Shopify is your strongest performer this month",
      metric: "$12,450 (45% of total revenue)",
      recommendation: "Increase Shopify inventory by 20%; Amazon underperforming",
      insightType: "channel_comparison"
    }
    
    Avoid: Sending raw metrics without context; duplicating data fetches (cache with React Query); generating insights for empty datasets
  </action>

# Phase 4 Plan 04: Alert Sensitivity Settings

**Vendor-configurable alert thresholds to reduce false positives and tune anomaly detection sensitivity per business need.**

## Overview

Vendors receive alert fatigue when sensitivity is too high. Plan 04 empowers vendors to adjust anomaly detection thresholds per type (Revenue Dip, Inventory Gap, Channel Mismatch, Performance Dip, Seasonal) on a 1-10 scale, with real-time preview of impact before committing changes.

## Architecture

```
Anomaly Detection (04-02)
    ↓
Alert Persistence (04-02)
    ↓
Alert Sensitivity Settings (04-04) ← ThresholdEngine filters alerts
    ↓
Alert Panel (04-02) ← Shows filtered alerts
    ↓
Vendor UI: AlertSensitivity page ← Manage thresholds
```

## Task Breakdown

### Task 1: Types & Schema (5min)
- Create `src/types/alert-sensitivity.ts` with SensitivityThreshold and SensitivitySettings
- Create migration `005_alert_sensitivity_schema.sql` with alert_sensitivity_settings table
- Add audit_updated_at trigger for history tracking

### Task 2: Threshold Engine (10min)
- Implement `src/lib/alert-sensitivity.ts` ThresholdEngine class
- Methods: `applySensitivity()`, `getSensitivityDefaults()`, `getRecommendedThreshold()`
- Add helper: `isAlertFiltered(alert, threshold)` for individual alert filtering

### Task 3: UI Pages (15min)
- Create `src/pages/AlertSensitivity.tsx` with 5 anomaly type sections
- Add range sliders (1-10) with label and current setting display
- Build `src/components/ThresholdPreview.tsx` showing filtered alert samples
- Add Save/Cancel buttons with React Query mutation for persistence

### Task 4: Integration (10min)
- Update `src/components/AlertPanel.tsx` to load sensitivity settings and filter alerts
- Update `src/lib/alert-engine.ts` to mark alerts with sensitivity context
- Update `src/pages/Dashboard.tsx` with link to AlertSensitivity page
- Verify alert count badge reflects filtered count

## Acceptance Criteria

- AC-1: Vendor can adjust sensitivity (1-10) per anomaly type and save preferences
- AC-2: Preview shows filtered alert count and sample alerts before save
- AC-3: AlertPanel displays filtered alerts respecting vendor sensitivity settings
- AC-4: Defaults are conservative (5-6) to minimize false positive fatigue
- AC-5: Threshold changes persist to database with audit trail

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Overly aggressive filtering hides real anomalies | Conservative defaults + preview before commit |
| Threshold calculation complexity | Use linear severity scoring (simpler than ML-based) |
| Performance on large alert histories | Client-side filtering (no DB re-query) |

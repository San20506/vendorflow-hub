<objective>

## Goal
Implement bulk edit and categorize operations for products, orders, and settlements with audit logging and preview.

## Purpose
After vendors import data, they need to correct and categorize records in bulk (e.g., "Mark 50 products as 'In Stock'", "Assign category to 100 uncategorized products"). This unlocks post-import data quality workflows without manual row-by-row editing.

## Output
- Bulk Operations page at `/bulk-operations`
- Bulk select UI (multi-select toolbar with record count)
- Bulk edit modal (choose field → enter value → preview → commit)
- Bulk categorize modal (assign category to multiple records)
- Supabase edge function for atomic batch updates
- Audit logging (who changed what, when, how many records)

</objective>

## AC-1: Bulk Selection Interface
```gherkin
Given a vendor on the Bulk Operations page
When they click "Select All" or checkboxes on individual records
Then the UI shows "X records selected" with clear selection count
And a toolbar appears with "Edit", "Categorize", "Clear" actions
And selection state persists when navigating between record types (products, orders, settlements)
```

## AC-2: Bulk Edit Modal
```gherkin
Given a vendor has selected records (e.g., 50 products)
When they click "Edit"
Then a modal opens with:
  - Field dropdown (e.g., "Status", "Category", "SKU")
  - Value input (text, select, or date depending on field type)
  - Preview showing "Will update X records with: [field] = [value]"
When they click "Update"
Then records commit to Supabase and modal closes
And UI shows success message: "✓ 50 records updated"
And audit log records: user_id, field, old_value, new_value, record_count, timestamp
```

## AC-3: Bulk Categorize Modal
```gherkin
Given a vendor has selected products missing categories
When they click "Categorize"
Then a modal opens with:
  - Category dropdown (existing categories from schema)
  - Preview showing "Will assign [category] to X uncategorized products"
When they click "Assign"
Then records update and modal closes
And UI shows: "✓ X products now have category [category]"
And audit log records the bulk categorize action
```

## AC-4: Audit Trail
```gherkin
Given any bulk operation (edit, categorize)
When the operation completes
Then an audit log entry is created with:
  - vendor_id, user_id, operation_type (edit/categorize)
  - field_name, new_value, record_count
  - timestamp, status (success/failure)
When a vendor visits "Audit" tab
Then they see list of recent bulk operations with details
```

## AC-5: Error Handling
```gherkin
Given a bulk operation that fails (e.g., Supabase connection error)
When the error occurs
Then vendor sees: "Update failed. Please try again later."
And dev logs capture: error detail, which records failed, retry hint
And operation can be retried without duplicate updates
```

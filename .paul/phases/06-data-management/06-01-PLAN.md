<objective>

## Goal
Implement a bulk data import system that allows vendors to drag & drop files (CSV/JSON/JSONL/XLSX), auto-detect schema, preprocess data via Gemini, and ingest into Supabase with real-time metrics.

## Purpose
Vendors need to bootstrap/backfill their data without manual entry. This unlocks historical reconciliation, corrections, and multi-entity uploads in one operation. Aligns with core goal: vendors control their data.

## Output
- Import page with drag & drop UI
- Data preprocessing pipeline (format detection → schema mapping → validation)
- Gemini-powered error fixing (96%+ confidence)
- Ingestion engine with metrics dashboard
- Dev error logging + vendor-friendly feedback
- Supabase edge function for async processing

</objective>

## AC-1: File Upload & Format Detection
```gherkin
Given a vendor on the Import page
When they drag & drop a CSV/JSON/JSONL/XLSX file

And previews: "Detected: 500 products, 200 orders, 50 settlements"
```

## AC-2: Auto-Detection & Field Mapping
```gherkin
Given a file with headers [product_id, name, price, category_id]
When the preprocessing pipeline runs
Then it auto-maps to schema fields without vendor input
And flags missing required fields: "category_id missing for 5 records"
```

## AC-3: Gemini Error Fixing
```gherkin
Given data with fixable errors (e.g., "price: 'abc'" → should be number)
When Gemini confidence ≥96%
Then the system auto-fixes and notes: "Fixed 12 type errors, confidence 98%"
When confidence <96%
Then it flags for dev review (no vendor-side error detail shown)
```

## AC-4: Metrics & Success Feedback
```gherkin
Given a successful import
When processing completes
Then vendor sees: "✓ 500 products, 200 orders, 50 settlements imported successfully"
And dashboard shows: import time, record counts, skipped entities
And no technical error details are visible to vendor
```

## AC-5: Error Handling & Dev Flagging
```gherkin
Given an import with >4% failure rate or unfixable errors
When processing completes
Then vendor sees: "Import encountered issues. Try again later."
And dev receives flagged log entry with full error context, file sample, and timestamp
```

## AC-6: Multi-Entity Single-File Import
```gherkin
Given a file containing products, orders, AND settlements
When the system processes it
Then it ingests each entity type to correct table
And reports per-entity metrics: "Products: 500 ✓, Orders: 200 ✓, Settlements: 50 ✓"
And skips incomplete entities: "Orders: 15 skipped (missing customer id)"
```

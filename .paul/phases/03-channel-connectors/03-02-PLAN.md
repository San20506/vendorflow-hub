<objective>
## Goal

Implement Amazon Selling Partner API (SP-API) authentication and product sync, enabling vendors to connect their Amazon Seller accounts and import catalog inventory into VendorFlow.

## Purpose

Amazon is the second-highest priority channel (after Shopify). Vendors selling on Amazon need to centralize inventory from their Seller Central account. This plan reuses the connector framework from Plan 01 — only the Amazon-specific OAuth and API integration is new.

## Output

- Amazon OAuth flow (SP-API authentication with automatic credential refresh)
- Amazon product catalog fetch (list all SKUs and inventory levels)
- AmazonConnector class (extends BaseConnector)
- Environment configuration for Amazon credentials (.env.local)
- Integration test to verify Amazon API connectivity
</objective>


## AC-2: Product Catalog Fetch from Amazon
```gherkin
Given Amazon is connected and authenticated
When vendor clicks "Sync Now" for Amazon channel
Then fetch all SKUs from Amazon Seller Central (including inventory levels)
And normalize to local ChannelProduct format (name, price, stock)
And insert/update in products table with (channel_id, external_product_id)
And handle pagination (Amazon returns max 20 results per page)
And show sync progress and completion status
```


       - 429 Throttle: Implement rate limiting (1-2 requests per second)
       - 401 Unauthorized: Trigger token refresh, retry

  <action>
    Implement hybrid OAuth approach:
    1. Vendor-initiated OAuth flow:
       - Vendor clicks "Connect Amazon" in Channels UI
       - Redirected to Amazon OAuth consent screen
       - After approval, exchange authorization code for tokens
    
    2. Token storage (in channels table):
       - Store access_token (expires in ~1 hour)
       - Store refresh_token (long-lived, ~5 years)
       - Store expires_at timestamp for access_token
    
    3. Automatic token refresh:
       - Before any API call, check if access_token expired
       - If expired, use refresh_token to obtain new access_token
       - Update channels table with new tokens and expiry
       - Retry original API call with fresh token
    
    4. Error handling for token refresh:
       - If refresh_token invalid (vendor revoked access), update sync_status to 'error'
       - Show user: "Amazon connection expired. Please reconnect."
       - Next sync attempt will fail until vendor re-connects
    
    Implementation in Task 1 (amazon-auth.ts):
    - generateAuthUrl(vendorId, redirectUri): string
    - exchangeAuthCode(code): Promise<{ access_token, refresh_token, expires_in }>
    - refreshAccessToken(refreshToken): Promise<{ access_token, expires_in }>
    - shouldRefreshToken(expiresAt): boolean
    
    BaseConnector.syncProducts() will auto-call refresh before fetchProducts() if needed.

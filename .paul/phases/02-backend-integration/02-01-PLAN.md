<task type="auto">
  <name>Design and create Supabase schema migrations</name>
  <files>supabase/migrations/001_init_schema.sql</files>
  <action>
    Create SQL migrations with:
    
    1. Users table (auth foundation):
       - id: UUID (primary key, auto-generated)
       - email: varchar unique
       - password_hash: varchar (bcrypt)
       - role: enum (admin, vendor, operations)
       - avatar: text (optional)
       - created_at: timestamp
       - updated_at: timestamp
    
    2. Vendors table:
       - vendor_id: UUID primary key
       - user_id: UUID (FK to users)
       - name: varchar
       - commission_rate: decimal
       - created_at: timestamp
    
    3. Products table:
       - product_id: UUID primary key
       - vendor_id: UUID (FK to vendors)
       - sku: varchar unique
       - name: varchar
       - price: decimal
       - stock: integer
       - created_at: timestamp
    
    4. Orders table:
       - order_id: UUID primary key
       - product_id: UUID (FK to products)
       - quantity: integer
       - total_amount: decimal
       - status: enum (pending, confirmed, shipped, delivered, cancelled)
       - created_at: timestamp
    
    5. Returns table:
       - return_id: UUID primary key
       - order_id: UUID (FK to orders)
       - reason: text
       - status: enum (requested, approved, rejected)
       - created_at: timestamp
    
    6. Settlements table:
       - settlement_id: UUID primary key
       - vendor_id: UUID (FK to vendors)
       - net_amount: decimal
       - status: enum (pending, processed)
       - created_at: timestamp
    
    7. Expenses table:
       - expense_id: UUID primary key
       - vendor_id: UUID (FK to vendors)
       - category: varchar
       - amount: decimal
       - created_at: timestamp
    
    Add indexes on FK columns and status enums.
    Avoid: Naming conflicts with Supabase system columns
  </action>
  <verify>
    1. Run: `supabase migration list` — migration appears in list
    2. Run: `supabase db push` — migration applies without errors
    3. Run: `psql $DATABASE_URL -c "\\dt"` — all tables visible
    4. Verify FK constraints: `psql $DATABASE_URL -c "SELECT * FROM information_schema.referential_constraints;"`
    5. Schema matches AC-1 requirements
  </verify>
  <done>AC-1 satisfied: Schema created with all required tables and relationships</done>
</task>
